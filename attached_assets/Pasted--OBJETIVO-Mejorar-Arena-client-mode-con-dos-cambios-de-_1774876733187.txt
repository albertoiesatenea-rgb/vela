## OBJETIVO
Mejorar Arena `client` mode` con dos cambios de alto valor didáctico:
1. añadir un `journey` visual rápido y entendible para ubicar al usuario en qué parte del proceso de venta está, qué falta y cuál es el siguiente paso;
2. añadir una vista alternativa minimalista centrada en la frase actual del vendedor, para que el usuario pueda enfocarse en lo que está diciendo ahora sin distracción del historial completo.

## CONTEXTO
- Arena ya tiene dos roles:
  - `seller`: el usuario vende y la IA hace de cliente
  - `client`: el usuario hace de cliente y la IA hace de vendedor
- En `client` mode el valor didáctico no está solo en ver qué responde el vendedor, sino en entender:
  - dónde estamos en el proceso,
  - qué intenta conseguir ahora,
  - qué debería venir después,
  - cuánto falta para un avance o cierre razonable.
- Copiloto ya usa una lógica de orientación con `journey` (`past / now / next`) y con bloques como `reading`, `mission`, `next_move`; quiero tomar esa lógica como referencia conceptual, pero adaptada a Arena de forma visual y no intrusiva. :contentReference[oaicite:0]{index=0} :contentReference[oaicite:1]{index=1}
- Además, la vista chat actual da demasiado detalle para ciertos usos. Quiero mantenerla, pero añadir una vista más minimalista donde la atención principal esté en la frase actual del vendedor, en grande, como si fuera una conversación más viva y menos “log de chat”.
- No quiero una solución ornamental. Quiero una mejora real de aprendizaje y de foco.

## RESTRICCIONES
- No tocar Copiloto
- No cambiar seller mode salvo reutilización visual segura de componentes
- No rehacer Arena desde cero
- No romper `start / turn / suggest / finish`
- No meter una timeline universal rígida ni una barra decorativa falsa
- No usar porcentajes arbitrarios de progreso
- No volver la UI intrusiva ni recargada
- No eliminar la vista chat actual; debe coexistir con la nueva vista
- No instalar dependencias nuevas salvo necesidad real
- Mantener el estilo minimalista de la app

## CAMBIO EXACTO

### PARTE 1 — JOURNEY VISUAL EN CLIENT MODE

Quiero añadir un `journey` híbrido:
- con una estructura funcional corta y estable,
- pero con contenido dinámico adaptado al caso real de venta.

#### Qué debe mostrar
En un vistazo, el usuario debe poder ver:
- **dónde estamos**
- **qué paso viene después**
- **qué falta hasta el avance o cierre**
- **si el vendedor está intentando cerrar demasiado pronto**

#### Estructura recomendada
No usar una timeline gigante.
Usar una secuencia corta de funciones comerciales, por ejemplo:
- Contexto
- Problema
- Bloqueo
- Encaje
- Avance
- Cierre

Pero la explicación concreta de cada paso debe adaptarse al caso real.

Ejemplo:
- visible: `Bloqueo`
- tooltip: “Aquí el vendedor intenta aislar qué frena de verdad la decisión.”
- ayuda del paso actual: “Ahora mismo el bloqueo es el cashflow mensual negativo.”
