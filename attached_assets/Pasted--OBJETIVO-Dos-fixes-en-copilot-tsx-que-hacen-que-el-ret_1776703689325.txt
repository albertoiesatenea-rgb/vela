## CONTEXTO
El retropass IA actual solo manda turnos UNKNOWN o baja confianza al modelo.
El problema es que turnos MAL clasificados (ej: Alberto hablando sale como
CLIENTE con confianza 0.4) no son candidatos y nunca se corrigen.

El modelo necesita ver el transcript completo para razonar correctamente
quién habla en cada turno — sin contexto de los turnos vecinos no puede
inferir el rol.

## CAMBIO REQUERIDO

En la función aiSpeakerRetropass en copilot.tsx:

ANTES — solo manda candidatos:
  const candidateTurns = log.filter(
    t => t.inferred_speaker === "UNKNOWN" || (t.speaker_confidence ?? 1) < 0.5,
  );
  if (candidateTurns.length === 0) return log;
  ...
  turns: candidateTurns.map(t => ({ index: t.turn_index, text: t.raw_fragment })),

DESPUÉS — manda TODOS los turnos al modelo pero solo aplica correcciones
en los que tienen baja confianza o son UNKNOWN:

  // Siempre mandar todos los turnos para que el modelo tenga contexto completo
  if (log.length === 0) return log;
  
  // Identificar cuáles son candidatos a corrección (baja confianza o UNKNOWN)
  const candidateIndices = new Set(
    log
      .filter(t => t.inferred_speaker === "UNKNOWN" || (t.speaker_confidence ?? 0) < 0.5)
      .map(t => t.turn_index)

  // Si no hay candidatos, no hay nada que corregir — pero igual vale la pena
