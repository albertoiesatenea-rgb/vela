1) Backend — nuevo endpoint POST /api/copilot/prebrief-context
Archivo: artifacts/api-server/src/routes/copilot.ts

Crea un endpoint que reciba:
{
  raw_input: string,
  has_asset?: boolean,
  call_type_hint?: string,
  user_risk_hint?: string
}

Debe devolver SOLO este JSON:
{
  "detected_phase": string,
  "call_type": string,
  "today_decision": string,
  "what_client_knows": string[],
  "main_blocker_probable": string,
  "valid_outcome_today": string,
  "confidence": "high" | "medium" | "low",
  "context_for_brief": string
}

- Leer el caso según fase real del proceso, no tratar todo como cierre ni todo como discovery.
- Detectar qué se decide hoy de verdad.
- Detectar qué sabe ya el cliente.
- Detectar el bloqueo principal probable.
- Detectar cuál sería un outcome válido hoy aunque no haya cierre.
- Compactar el resultado para poder confirmarlo antes de generar el briefing.
- No inventar cifras, hechos ni decisiones del cliente.
- Responder SOLO JSON válido.
- Idioma: español.
- Tono: directo, táctico, compacto.

Añade tracking con logAICall usando:
