FIX A — Splitter más agresivo
─────────────────────────────────────────

## CONTEXTO
En speaker-session.ts, la función splitIntoMiniTurns tiene estos parámetros:
  MIN_SEG = 22
  MAX_SEGS = 5

El problema: bloques de 400-800 chars llegan sin separar porque los marcadores
de Tier 2 requieren MIN_SEG*2 = 44 chars de contexto previo, y muchos
cambios de speaker ocurren en posiciones donde no hay marcadores claros.

Los marcadores más frecuentes en español conversacional de ventas que faltan:
- "claro" solo al inicio de respuesta
- "entendido" — casi siempre vendedor respondiendo
- "perfecto" — casi siempre vendedor
- "vale" solo como respuesta corta (cliente)
- "y" al inicio de frase tras pausa
- preguntas directas: "¿tienes...?", "¿puedes...?", "¿cuánto...?"

## CAMBIOS REQUERIDOS

En splitIntoMiniTurns, en la sección Tier 2, añadir estos marcadores
adicionales con offset T2 (MIN_SEG * 2):

  mark(/\bperfecto\b/g, T2);        // vendedor confirmando
  mark(/\bentendido\b/g, T2);       // vendedor procesando
  mark(/\bde acuerdo\b/g, T2);      // acuerdo/confirmación
  mark(/\baproximadamente\b/g, T2); // vendedor preguntando datos
  mark(/\bcuánto\b/g, T2);         // pregunta — casi siempre vendedor
  mark(/\btienes\b/g, T2);         // pregunta directa — vendedor
  mark(/\bpodrías\b/g, T2);        // pregunta — vendedor
  mark(/\bel primer\b/g, T2);      // vendedor estructurando
  mark(/\bel segundo\b/g, T2);
  mark(/\bel tercer\b/g, T2);
  mark(/\bpor cierto\b/g, T2);

Además, en la sección de punctuation-based splits, ampliar la lista de
response starters que disparan split tras [.?!]:

  ANTES:
  if (/^(sí|no|claro|pues|mira|bueno|hola|oye|vale|a ver|exacto|perfecto|...)/.test(next))

  DESPUÉS — añadir al inicio del regex:
  "entendido|de acuerdo|aproximadamente|cuánto|podrías|tienes|el primer|
   el segundo|el tercer|nosotros|nuestro"

También aumentar MAX_SEGS de 5 a 8 para permitir más splits en bloques largos.
