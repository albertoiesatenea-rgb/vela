Quiero hacer un refactor completo y ordenado del sistema de guardado + log forense canónico de VELA en Copiloto. NO hagas un parche rápido. Usa la arquitectura actual y deja una única fuente de verdad para el log descargable.

OBJETIVO GLOBAL

Quiero que una sesión de Copiloto se pueda:
1. cerrar sin guardar en DB
2. guardar explícitamente en DB
3. descargar como un ÚNICO markdown forense canónico
4. volver a descargar EXACTAMENTE el mismo archivo desde la pantalla de historial
5. auditar después en ChatGPT o Claude con toda la información comercial y técnica del sistema

REQUISITO CLAVE

El archivo descargado desde la pantalla post-llamada y el archivo descargado desde historial deben ser EXACTAMENTE el mismo markdown canónico.
Prohibido tener dos builders distintos o dos versiones distintas del log.
Debe existir un único builder canónico del log en frontend, y el markdown generado por ese builder debe persistirse en DB como canonicalLogMd.
Historial debe descargar canonicalLogMd tal cual desde DB, sin reconstruir otra versión.

REGLAS DURAS

- No borres funcionalidad actual útil.
- No cambies la doctrina comercial ni los prompts de brain salvo lo estrictamente necesario para la fuente de transcript post-llamada.
- No metas hardcodes raros para “Amara” ni similares.
- No inventes razonamientos ocultos. Si explicas por qué VELA dijo algo, usa solo datos observables: fragmento, memoria, contexto, reading, mission, next_move, etc.
- Todo análisis post-llamada debe usar como fuente principal el transcript limpio de Whisper con atribución de hablantes.
- Web Speech queda como fuente secundaria / fallback explícito, nunca como fuente silenciosa.
- Si Whisper limpio todavía no está listo al terminar llamada, summary / score / brutal audit / VELA audit deben esperar hasta 45s máximo. Si aun así no está, deben entrar en “modo degradado” y dejarlo explícito en UI y en el log.
- Mantén el diseño coherente con el resto de la app.
