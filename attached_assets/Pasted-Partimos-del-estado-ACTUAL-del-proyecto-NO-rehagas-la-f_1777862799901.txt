C) PREBRIEF-SCRIPT — CHECKPOINT PREVIO ANTES DE PROPUESTA
Refuerza prebriefScriptRules y guardrails para que, en casos tipo Antonio:
- real_call_goal NO sea simplemente “ver si pasa a propuesta”

  “confirmar si el caso supera el checkpoint estructural y ordenar el criterio dominante antes de propuesta”

  2. aterrizar el freno estructural

  5. cerrar docs + fecha + siguiente conversación
- suggested_opening debe sonar más afilada y menos plantilla
- suggested_next_step_close debe usar el patrón:
