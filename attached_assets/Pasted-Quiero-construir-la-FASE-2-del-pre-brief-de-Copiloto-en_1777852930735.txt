2) CALIDAD DEL ENDPOINT

Quiero que este briefing sea táctico y corto.

Reglas:
- nada de teoría de ventas
- nada de texto inflado
- nada de “depende” salvo que sea imprescindible
- usar el brain activo
- usar el contexto ya interpretado como fuente principal
- no inventar datos
- no mezclar 8 objeciones: máximo 3 objeciones esperadas
- no generar un guion robot
- el objetivo es preparar una llamada real

Qué significa cada campo:

- `real_call_goal`
  = cuál es el objetivo comercial real de ESTA llamada, no el ideal abstracto

- `must_get_today`
  = 3-5 cosas que el vendedor necesita conseguir hoy para que la llamada haya merecido la pena

- `expected_objections`
  = máximo 3
  Cada una con:
  - objeción probable
  - por qué probablemente aparecerá en este caso
  - cómo manejarla tácticamente, sin discurso largo

- `mistakes_to_avoid`
  = máximo 5 errores concretos del vendedor en este caso

- `suggested_call_structure`
  = secuencia breve de 4-6 pasos
  Ejemplo:
  - abrir y encuadrar
  - validar objetivo
  - aislar freno
  - resolver criterio clave
  - cerrar siguiente paso

- `suggested_opening`
  = una apertura útil, natural, no demasiado larga

- `suggested_next_step_close`
  = una frase de cierre orientada al siguiente paso realista de esa fase

- `brief_for_live`
  = bloque compacto de contexto táctico listo para pasar a Copiloto live
  Debe integrar:
  - fase
  - objetivo real
  - freno dominante
  - qué conseguir hoy
  - siguiente paso esperado
  En 4-7 frases máximo
